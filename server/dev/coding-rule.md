# コーディング規約

## はじめに

サーバサイドは原則としてPythonで構築する。  
APIフレームワークとしてFastAPIを採用する。  
選定理由は軽量かつ信頼性があり、さらにはテストがしやすいため。  

## インデント

4文字のソフトタブを使用する。  
これは[PEP8](https://pep8-ja.readthedocs.io/ja/latest/#section-4)の通り。  

## 1行の長さ

[PEP8](https://pep8-ja.readthedocs.io/ja/latest/#section-6)に従い、プログラムコードは最大79文字、コメント(ドキュメンテーション文字列含む)は72文字以内に収めることとする。  
これを超えるプログラムコードを記述する際には、「\」を使用するのではなく、かっこを用いて実現する。  

## モジュール戦略

原則として、「app」ディレクトリをルートとして、コンポーネントはそれ配下のディレクトリに格納する。  
また、それらをインポートする際には、「*」を使用せず、全てを明示的に列挙することとする。  

```py
# 良い例
from mod import a, b, c
```

```py
# 悪い例
from mod import *
```

## メソッドチェーン(関数チェーン)

テスト容易性の観点からメソッドチェーンは原則として使用せず、ひとつひとつの処理ごとに変数に格納する。  
これはメモリ効率化の観点からは推奨されないが、テスト容易性のためにこれを許容することとする。  

```py
# 良い例
a = add(1, 2)
b = a.execute()
c = b.successed
```

```py
# 悪い例
c = add(1, 2).execute().successed
```

## 引用符

原則としてシングルクォーテーションを使用する。  
例外として文字列内にシングルクォーテーションを含む場合には不要なエスケープを避けるため、ダブルクォーテーションを採用する。  

## 変数名

原則として、[PEP8](https://pep8-ja.readthedocs.io/ja/latest/#section-20)の通り。  

- クラス名 - パスカルケース
- 関数名 - スネークケース
- 変数名 - スネークケース
- 定数名 - スネークケース(全て大文字)
